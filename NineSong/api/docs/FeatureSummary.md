# 媒体库音频文件管理功能总结与最佳实践

本文档总结了媒体库音频文件上传下载功能的完整实现，包括架构设计、关键特性、使用指南和最佳实践。

## 功能概述

媒体库音频文件管理功能是一个完整的文件同步解决方案，支持用户上传、下载和管理音频文件。该功能采用Clean Architecture架构，确保了良好的可维护性和可扩展性。

### 核心功能
1. **音频文件上传**: 支持单文件和分块上传，适用于各种文件大小
2. **音频文件下载**: 安全高效的文件下载机制
3. **进度监控**: 实时跟踪上传/下载进度
4. **文件管理**: 支持按媒体库和上传者分类管理文件
5. **同步记录**: 完整的操作审计和状态跟踪
6. **元数据提取**: 自动提取音频文件的元数据信息
7. **元数据同步**: 支持封面图片、歌词文件和标签信息的同步上传下载
   - **歌词格式支持**: 支持 LRC、TXT、KRC（酷狗）、QRC（QQ音乐）等多种歌词格式
   - **封面图片**: 支持 JPEG、PNG 格式，自动生成多尺寸版本
   - **标签信息**: 支持自定义标签、心情标签、场景标签的同步

## 架构设计

### Clean Architecture 层次
```
┌─────────────────┐
│   Controllers   │ ← HTTP请求处理
├─────────────────┤
│    Usecases     │ ← 业务逻辑处理
├─────────────────┤
│   Repositories  │ ← 数据访问层
├─────────────────┤
│     Domain      │ ← 业务实体和规则
└─────────────────┘
```

### 数据流
1. 客户端发起请求 → Controller 解析请求
2. Controller 调用 Usecase 执行业务逻辑
3. Usecase 通过 Repository 访问数据
4. Repository 操作 MongoDB 数据库
5. 返回结果给客户端

### 关键组件

#### 1. MediaLibraryAudio 实体
- 存储音频文件的基本信息、元数据和状态
- 包含文件路径、大小、类型、校验和等信息
- 支持艺术家、专辑、时长等音频元数据

#### 2. 同步记录系统
- MediaLibrarySyncRecord 跟踪每个文件操作的状态
- 支持上传、下载、同步等多种操作类型
- 提供完整的操作审计轨迹

#### 3. 分块上传机制
- ChunkedUploadSession 管理大文件的分块上传
- 支持断点续传和进度监控
- 适用于大文件的可靠传输

## API端点汇总

### 音频文件管理
- `POST /media-library-audio/upload` - 上传音频文件
- `GET /media-library-audio/download/:file_id` - 下载音频文件
- `GET /media-library-audio/info/:file_id` - 获取文件信息
- `GET /media-library-audio/library/:library_id` - 获取指定库文件列表
- `GET /media-library-audio/uploader/:uploader_id` - 获取指定用户文件列表
- `DELETE /media-library-audio/:file_id` - 删除音频文件

### 进度监控
- `GET /media-library-audio/progress/upload/:upload_id` - 获取上传进度
- `GET /media-library-audio/progress/download/:file_id` - 获取下载进度

## 技术特性

### 1. 性能优化
- **分块上传**: 对大文件进行分块处理，避免内存溢出
- **进度监控**: 实时反馈上传/下载进度，提升用户体验
- **文件校验**: MD5校验和确保文件完整性
- **索引优化**: 为常用查询字段建立数据库索引

### 2. 安全性
- **认证授权**: 使用JWT令牌验证用户身份
- **文件类型验证**: 严格验证上传文件的类型和格式
- **访问控制**: 基于用户权限控制文件访问
- **数据加密**: 敏感数据传输加密

### 3. 可靠性
- **错误处理**: 完善的错误处理和恢复机制
- **事务保证**: 确保数据操作的一致性
- **备份机制**: 数据库备份和恢复策略
- **监控告警**: 系统健康状态监控
