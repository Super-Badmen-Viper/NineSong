# 历史功能与路线图参考

这个文档用于保留旧版公开 README 里更完整的功能说明和路线图信息。

## 旧版 README 中已经列出的音乐场景能力

- 丰富的单层排序，以及多层混合排序与筛选
- 更深入的复合标签处理能力，用于强化音乐之间的关联
- 搜索跳转优化
- 面向标题、专辑、艺术家、歌词的模糊搜索
- 对中文拼音、简繁中文混合匹配的支持
- 相似搜索结果推荐
- 多种播放样式，例如方形封面、旋转封面、Beaut、Base、专辑列表样式等
- 针对不同音乐文件场景的专属播放模式
- 面向 CUE 的播放与文件管理工作流
- CUE 的可视化虚拟轨道播放
- 更完整的 TAG 导入能力，包括 `m4a` 和 `cue` 相关场景
- 基于标签云和兴趣偏好的第一阶段推荐能力
- 基于用户使用数据的轻量推荐第二阶段能力

## 旧版 README 中列出的后续音乐场景规划

- 虚拟列表与分页列表并存的双页浏览模式
- 客户端与服务端之间的文件上传、下载与同步
- 支持远程上传、自动关联、手动合并的可视化标签管理
- 更丰富的标签字段，例如艺术家头像、艺术家照片、专辑封面、音质版本、歌词版本
- 面向 ISO 的播放与文件管理能力
- 更丰富的音效能力，包括多声道效果和多档 EQ
- 可选的公益在线 TAG API 同步能力
- 基于音乐知识图谱的第三阶段推荐
- 结合知识图谱与 LLM 的第四阶段推荐

## 旧版 README 中描述的更大场景方向

旧版 README 还把 NineSong 描述为一个更广义的多媒体与个人数字中心方向，覆盖：

- 音频
- 视频
- 图片
- 文本
- 文档
- 压缩包
- 可执行文件
- 数据库
- 流媒体音乐与卡拉 OK 场景
- AI 模型部署场景
- 智能相册场景
- 影视中心场景
- 在线笔记场景
- 文档工作台场景
- 电子书阅读器场景
- 知识图谱推荐
- 国际化
